  如果你使用 Claude Code 开发多个项目，每个项目需要不同的 Node.js 版本，CVM 是最佳选择：

  - ✅ **干净隔离** - 版本存储在项目目录中，避免全局污染
  - ✅ **Claude Code 集成** - 专为 Claude Code 工作流设计，支持 `node_modules` 和 `.claude` 的链接文件夹

- [ ] **Step 6: Translate "Commands" section**

  ```markdown
  ## 命令

  ### `cvm setup`

  在当前工作区目录初始化 CVM。创建 `versions/` 文件夹和 shim 脚本。

  ### `cvm install [version]`

  安装指定 Node.js 版本或从 `package.json` 读取版本：

  ```bash
  cvm install        # 安装 package.json 中 engines.node 指定的版本
  cvm install 20     # 安装最新 20.x.x
  cvm install 20.11  # 安装最新 20.11.x
  cvm install 20.11.0 # 安装确切版本
  ```

  ### `cvm uninstall <version>`

  从工作区移除已安装的 Node.js 版本：

  ```bash
  cvm uninstall 20.11.0
  ```

  ### `cvm use [version]`

  切换当前 shell 会话使用的 Node.js 版本：

  ```bash
  cvm use        # 使用 package.json 版本
  cvm use 20.11.0 # 使用指定版本
  ```

  ### `cvm current`

  显示当前激活的 Node.js 版本：

  ```bash
  cvm current
  ```

  ### `cvm list`

  列出工作区中已安装的所有版本：

  ```bash
  cvm list
  ```

  ### `cvm patch proxy`

  在 `node_modules/.bin` 中的每个 CLI 脚本周围安装代理补丁：

  ```bash
  cvm patch proxy
  ```

  这让脚本在运行前自动激活正确的 Node.js 版本。

  ### `cvm patch revert`

  移除所有代理补丁并恢复原始脚本：

  ```bash
  cvm patch revert
  ```

  ### `cvm patch status`

  检查当前工作区中代理补丁的状态：

  ```bash
  cvm patch status
  ```
  ```

- [ ] **Step 7: Translate "How It Works" section**

  ```markdown
  ## 工作原理

  ### 目录结构

  CVM 在项目目录下维护一个 `versions/` 文件夹：

  ```
  my-project/
  ├── .claude/
  ├── src/
  ├── package.json
  └── versions/
      └── v20.11.0/
          └── contents/
              ├── bin/
              │   ├── node
              │   ├── npm
              │   ├── npx
              │   └── ...
              └── ...
  ```

  ### Shim 机制

  CVM 使用 "shim" 脚本作为 `node`、`npm`、`npx` 等命令的入口点：

  1. **Shim 脚本** - 调用时，确定应使用哪个版本的 Node.js
  2. **版本激活** - 从当前工作目录向上查找 `package.json`
  3. **执行** - 将命令转发到正确的 Node.js 二进制文件

  这意味着：
  - ✅ 你不需要手动切换版本
  - ✅ 每个项目自动获得正确的 Node.js
  - ✅ 全局 `node` 命令只是垫片（shim）

  ### 代理补丁

  对于 `node_modules/.bin` 中的 CLI 工具：

  - **代理脚本** - 包装原始 CLI 脚本
  - **激活** - 在执行前激活正确的 Node.js 版本
  - **透明** - 无需改动即可工作

  这确保像 `vite`、`tsc`、`eslint` 这样的工具都使用项目指定的 Node.js 版本。
  ```

  ## 配置

  通过 `package.json` 配置 CVM：

  ```json
  {
    "name": "my-project",
    "engines": {
      "node": "20"
    },
    "cvm": {
      "autoInstall": true
    }
  }
  ```

  ### 选项

  - `engines.node` - 指定所需的 Node.js 版本（语义化版本范围）
  - `cvm.autoInstall` - 如果为 true，版本不存在时自动安装
